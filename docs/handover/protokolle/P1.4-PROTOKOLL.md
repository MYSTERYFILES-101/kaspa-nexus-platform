# P1.4-PROTOKOLL: Token Detail Seite

**Datum:** 2025-12-03
**Entwickler:** Claude (AI Assistant)
**Sprint:** P1 - Core Frontend

---

## Aufgabe

Implementierung der Token Detail Seite (`/krc20/tokens/[ticker]`) mit vollstandigem Token-Profil, Statistiken, Chart-Placeholder und Holder-Distribution.

---

## Implementierte Features

### 1. Route & Navigation
- **Route:** `/krc20/tokens/[ticker]` (z.B. `/krc20/tokens/NACHO`)
- **TokenCard Link:** Korrigiert von `/tokens/` zu `/krc20/tokens/`
- **Back-Navigation:** Link zurueck zur Token-Liste

### 2. Token Header
- Token Icon (Logo oder Fallback mit ersten 2 Buchstaben)
- Token Name und Ticker
- Live-Preis in USD
- 24h Preisaenderung mit farbiger Anzeige
- Rang-Badge (#1, #2, etc.)
- Preis in KAS
- Live-Indikator mit letzter Aktualisierungszeit

### 3. Market Statistics Grid
- Market Cap
- 24h Volume
- Holders
- Circulating Supply (Minted)
- Max Supply
- Transfers

### 4. Price Chart (Placeholder)
- TradingView Chart Placeholder
- Hinweis auf kommenden interaktiven OHLC Chart

### 5. Token Information
- Mint Progress Bar mit Prozentanzeige
- State (minting/finished/deployed)
- Decimals
- Mint Limit
- Total Mints
- Deployed At

### 6. Quick Stats (Sidebar)
- 24h Change
- 7d Change
- Price in KAS
- Rank

### 7. Top 10 Holders
- Rang
- Adresse (gekuerzt)
- Prozentanteil
- Hinweis auf 24h Update-Zyklus

### 8. External Resources
- Link zu kas.fyi
- Link zu Kaspa Explorer

### API Integration
- Endpoint: `/api/krc20/tokens/[ticker]`
- Backend: `api.kaspa-nexus.io/v1/krc20/tokens/[ticker]`
- Revalidate: 30 Sekunden
- Auto-Refresh: alle 30 Sekunden

### TypeScript Types verwendet
```typescript
Krc20TokenWithPrice  # Token mit Preisdaten
TokenHolder          # Holder-Informationen
```

### Mock Data
Da die Datenbank aktuell leer ist, zeigt die Seite Mock-Daten wenn kein Token gefunden wird:
- Beispiel-Preis, Volume, Market Cap
- 10 Mock-Holder mit Beispieladressen
